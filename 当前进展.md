## 已验证修复点

本轮 20+20 中已经纳入以下修复：

1. 短答案抽取与评分归一
   - `Final answer:` 优先。
   - 支持实体子串匹配：例如 `Dark Fantasy Choco Fills` 可按 gold `dark fantasy` 计为实体匹配。
   - 支持 ChartQA 平均值/估读类 relaxed numeric tolerance：`abs error <= 0.1`，例如 `21.7` 可归一为 `21.6`。
   - 支持截断年份修复：模型解释中出现 `2010年至2011年`，但 `Final answer: 201` 被截断时，可恢复为 `2011`。
   - 支持截断 Yes/No 比较题兜底：如果回答包含两个求和值，例如 `185` 与 `233`，但未输出 final answer，可按问题中的 greater/less 关系恢复 `Yes` 或 `No`。

2. 局部高清 crop
   - OCR bbox 写入 manifest/detail。
   - DocVQA 支持数字/年份 target crop、时间表 row crop、广告/商品/品牌 crop。
   - 广告/服饰题会优先裁包含品牌 logo 的区域，例如 `WILLS LIFESTYLE`。
   - MM-RAG top-k 排序会参考 region metadata score，保证 `--max-images 1` 时更优先传高价值 crop。

3. ChartQA 专用 route
   - 固定生成 `chart_region`。
   - 支持 count、difference、compare、min/max、average 等类型提示。
   - 当前 20 个 ChartQA 样本 MM-RAG 全部正确。

## 当前剩余 MM-RAG 错误

### `docvqa_49181`

- 问题：`What session is at 2.00 to 5.00p.m.?`
- Gold：`trrf scientific advisory council meeting`
- MM-RAG prediction：`TRRF 一般会议（第一部分`
- EM：0.0，ANLS：0.0
- Top evidence：`docvqa_49181_p1_region_keyword`

分析：

- 这是一道 DocVQA 日程表/会议表格行定位题。
- Text-RAG 在这题中答对了 `TRRF Scientific Advisory Council Meeting`，说明 OCR 文本里存在正确答案。
- MM-RAG 送入的 region crop 对应了错误或不完整的表格上下文，VLM 根据局部视觉区域读成了另一个 session。
- 当前问题不是短答案抽取，而是 region selection 对“时间 -> 对应表格行 -> session 名称”的结构化定位还不够稳。

下一步建议：

- 增强 DocVQA schedule/table row crop：
  - 对含时间范围的问题，优先匹配 `2.00`、`5.00`、`p.m.` 等 OCR box。
  - 以时间 box 为 anchor，横向扩展整行，保留左侧时间列和右侧 session 列。
  - 如果 OCR text chunk 与 visual crop 冲突，可让 text evidence 参与 MM-RAG top evidence，避免视觉 crop 错行。
- 对 schedule/table 问题增加结构化 OCR row reconstruction，把同一 y-band 的文本合并成候选行，再优先检索候选行。

## 结果解读

当前 pipeline 已经把主要错误从“答案格式/局部视觉缺失/图表 route 不稳定”推进到了更细的结构化表格定位问题。

现阶段效果可以概括为：

- ChartQA：在当前 20 样本上已经稳定到 20/20。
- DocVQA 普通实体、广告、数字、计数、时间类题目大部分已经稳定。
- DocVQA 剩余风险集中在复杂表格/日程表的行列对齐。
- Text-RAG 仍明显弱于 MM-RAG，尤其 ChartQA 几乎无法依靠纯 OCR 文本完成视觉读数。
